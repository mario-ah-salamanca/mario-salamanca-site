# CONTENT_STRATEGY.md

## Purpose

This document defines the content strategy for Mario Salamanca’s personal website.

The goal of the content system is to build long-term authority, trust, and opportunity by publishing useful, honest, technically grounded content around software engineering, systems thinking, workflows, AI, infrastructure, cybersecurity, creativity, and personal growth.

Content should support the website’s role as:

* A professional portfolio
* A consulting/freelance lead generator
* A career growth asset
* A founder credibility platform
* A public record of learning and execution

---

## Content Mission

Create content that shows how Mario thinks, builds, learns, and solves problems.

The content should help visitors understand:

* What Mario can build
* How Mario approaches technical problems
* What Mario is learning
* What kind of opportunities Mario is suited for
* Why Mario is trustworthy
* How Mario turns messy ideas into structured systems

## Content Types

### Case Studies

Purpose:

Prove capability.

Structure:

1. Title
2. One-line outcome
3. Context
4. Problem
5. Constraints
6. Role
7. Approach
8. Architecture
9. Technologies
10. Result
11. Lessons learned
12. Next improvements

Use for:

* Aerospace test tooling
* ETL/data modeling
* HMI/CAN work
* Personal website system
* AI-ready documentation kit

### Build Logs

Purpose:

## Content Tone

Content should be:

* Clear
* Practical
* Honest
* Reflective
* Technically grounded
* Human
* Useful

Content should not be:

* Overly academic
* Buzzword-heavy
* Fake expert positioning
* Too motivational
* Too casual
* Too long without structure
* Too focused on tools without explaining problems

---

## Editorial Rules

When writing content:

* Start with the problem.
* Explain why it matters.
* Use concrete examples.
* Show tradeoffs.
* Be honest about uncertainty.
* Do not invent results.
* Do not reveal confidential work details.
* Prefer clarity over cleverness.
* End with a useful takeaway or next step.

## Content Quality Checklist

Before publishing, ask:

1. Does this help the right audience?
2. Does this build trust?
3. Does this show how Mario thinks?
4. Is the title clear?
5. Is the intro specific?
6. Is there a useful takeaway?
7. Is there a next step?
8. Is it honest?
9. Is it connected to the rest of the site?
10. Would this still support the brand in 5 years?

If not, revise before publishing.
